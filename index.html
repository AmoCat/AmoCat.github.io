Morris先序及后序遍历

Veröffentlicht am 2016-08-11

二叉树的Morris先序及中序遍历

Morris先序遍历的过程如下：  
1. 如果当前节点的左孩子为空，则输出当前节点并将其右孩子作为当前节点。(没左子树)  
2. 如果当前节点的左孩子不为空，在当前节点的左子树中找到当前节点在中序遍历下的前驱节点。  
    a) 如果前驱节点的右孩子为空，将它的右孩子设置为当前节点。输出当前节点（在这里输出，这是与中序遍历唯一一点不同）。当前节点更新为当前节点的左孩子。（第一次到当前节点，应访问左子树）  
    b) 如果前驱节点的右孩子为当前节点，将它的右孩子重新设为空。当前节点更新为当前节点的右孩子。（访问完了cur的左子树，第二次到当前节点，应回退到当前节点）  
3. 重复以上1、2直到当前节点为空.

先序遍历的思想是，输出根节点后，遍历左子树后再遍历右子树。因此我们从A出发，扫荡完左子树后，一定会回到A。
既然我们为O（1）的空间复杂度，那么就关键点就在于在最底层的儿子B处，该回退到哪个祖先节点（一步回退到爹或者爷爷等祖先~）.Morris的巧妙之处在于:在我们第一次遍历节点A时，就会先去找到B。并用B的右儿子做标记，指向A，这样A到B之间一定有一个环，它们的相对顺序和先序遍历一模一样.
比如下图中，6245，21都组成了一个环。当我们第二次到A时（遍历完左子树），又会去掉这些“标记”，因为左子树的标记对我们没用了。如此反复，直到当前节点为空。

只要能够方便的到达A时是否遍历完了左子树，那么无论先序还是中序，就都很容易了。
以LeetCodeBinary Tree Inorder Traversal的中序遍历为例：

class Solution {
public:
    vector<int> inorderTraversal(TreeNode* root) {
        TreeNode *cur = root;
        TreeNode *pre = nullptr;
        vector<int> res;
        while(cur!=nullptr){
            if(cur->left==nullptr){
                res.push_back(cur->val);
                pre=cur;
                cur=cur->right;
            }else{
                TreeNode *p = cur->left;
                while(p->right !=nullptr && p->right != cur)
                p=p->right;
                if(p->right == nullptr){//第一次遍历到cur,应建标记并扫描左子树
                    p->right=cur;
                    cur=cur->left;
                }else{//第二次到cur，消除标记
                    res.push_back(cur->val);//与先序代码唯一不同的地方
                    p->right = nullptr;
                    cur=cur->right;
                }
            }
        }
        return res;
    }
};

杂七杂八的学习笔记（Python requirements.txt)

Veröffentlicht am 2016-08-10

Python 中的 requirments.txt 可以方便我们快速安装该项目所需要的类库 ，使用命令为 ：

pip install -r requirements.txt

当然，首先你的电脑里要安装了pip,有时需要sudo。

requirements.txt的生成方法有两个：freeze命令和pipreqs。
它们的区别是：Freeze 生成 Python环境下的所有类库到requirments.txt,而pipreqs只生成当前项目下中用到的类库。用法为：

pip freeze > requirements.txt

或

pip install pipreqs (已安装pipreqs的省略这步)

pipreqs  path

Veröffentlicht am 2016-08-10

在安装Theano时遇到了这样的错误：

Found existing installation: six 1.4.1  
DEPRECATION: Uninstalling a distutils installed project (six) has been deprecated and will be     removed in a future version. <span style="font-    family: Arial, Helvetica, sans-serif;">This is due     to the fact that uninstalling a distutils project     will only partially uninstall the project.</span>  
Uninstalling six-1.4.1:

经排查，并不是没有权限的问题，后来在git上发现一个类似的问题，找到原因：OS X EI Capitan本身已经安装了six 1.4.1 ,而安装theano时依赖于比1.4.1更高的six版本。OS X EI Capitan 自带的six在Apple中一个特定目录里，theano安装的six应安装到site-packages目录中，由于系统完整性保护（System Integrity Protection），pip不能卸载系统自带的six 1.4.1（即使有root权限也不能）只会跳过卸载,site-packages中也不会安装新的six版本。因此便出了以上错误。

git 类似问题参考https://github.com/pypa/pip/issues/3165

解决方法是：sudo pip install theano –upgrade –ignore-installed six

此方法可安装theano不会报错，但最终电脑里的six 仍是 1.4.1版本

从错误即可看出还是six版本问题。然而，机智的我又找到了类似问题哈哈哈，参考链接ImportError: No module named cPickle以及OS X不能upgrade six

综合两个链接里的办法，主要是下面几个：

  • 先卸载six，再安装，卸载不了就去System/Library/Frameworks/Python.framework/Versions/2.7/Extras/lib/python/ 手动删。

  • pip install –ignore-installed six 或者sudo pip install -U six经尝试，没有用（OS X EI Caption）

  • sudo easy_install –upgrade six 解决！前两次一直超时不知道为什么，最后成功了，此时在Python中import theano就可以不报错了。上面链接里也有人用2方法成功的，建议这几个命令都可以试一下~

总结一下，如果mac下因为six 1.4.1 的问题安装不了theano的话，就用下边两条命令：

sudo easy_install --upgrade six

sudo pip install  theano
